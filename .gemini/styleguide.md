# Gemini Code Assist レビューガイドライン

## 言語設定

- **必ず日本語で返信してください。**

## レビュー方針

- **品質向上**: コードの細部まで確認し、潜在的なバグだけでなく、可読性・命名・構造面のローレベルな問題点も積極的に指摘してください。
- **改善のしやすさ**: マジックナンバー、命名規則の違反、些細なパフォーマンス改善など、小さな問題もすべてコメントとして残してください。将来のリファクタリングを容易にする設計改善も積極的に提案してください。

## フォーカスエリア

- **正確性 (correctness)**: ロジックの誤り、エッジケースの未処理、型の不一致を検出してください。
- **効率性 (efficiency)**: 不要なループ、冗長な処理、パフォーマンスに影響する箇所を指摘してください。
- **保守性 (maintainability)**: 可読性の低いコード、重複コード、過度に複雑な構造を指摘してください。
- **セキュリティ (security)**: インジェクション、機密情報の露出、不適切なパーミッションなどの脆弱性を検出してください。

## PRサマリー

- PR がオープンされた際には、変更内容の要約を日本語で提供してください。
