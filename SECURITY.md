# Security Policy

## サポート対象バージョン

本リポジトリは `main` ブランチを唯一のサポート対象とします。リリースタグや過去ブランチへの修正バックポートは行いません。

## 脆弱性の報告

セキュリティ上の脆弱性を発見された場合は、**公開 Issue では報告せず**、以下のいずれかでプライベートに連絡してください。

- GitHub の **Private Vulnerability Reporting** 機能 (推奨)
  - リポジトリの `Security` タブ → `Report a vulnerability` から
- メール: `genzouw@gmail.com`

### 報告に含めていただきたい情報

- 脆弱性の概要と影響範囲
- 再現手順 (PoC が望ましい)
- 発見した環境 / バージョン (該当する場合は commit SHA)
- 提案する修正方針 (任意)

## 対応 SLA

個人運営のため確約はできませんが、目安として以下を目標とします。

| 重大度       | 初回応答      | 修正リリース |
| ------------ | ------------- | ------------ |
| Critical     | 3 営業日以内  | 7 日以内     |
| High         | 5 営業日以内  | 14 日以内    |
| Medium / Low | 10 営業日以内 | best-effort  |

## 安全な情報共有

報告のやり取りで秘匿が必要な情報を送る場合は、メール本文への直接記載を避け、GitHub Private Vulnerability Reporting のスレッドを利用してください。PGP 等の追加暗号化が必要な場合は、最初のメールでご相談ください。

## 想定外の対象

以下は本ポリシーの対象外です。

- 本リポジトリを fork した派生プロジェクト (各派生先に直接報告してください)
- 第三者サービス (Cloudflare / GCP / Neon / Stripe / Resend / LINE) 自体の脆弱性
  → 各サービスの脆弱性報告窓口へ
- 報告者自身のアカウント / 環境固有の設定ミスに起因するもの
