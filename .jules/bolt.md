## 2025-07-22 - Optimize expensive environment variable parsing in critical paths
**Learning:** Frequent array/set initialization or string splitting based on environment variables inside middleware functions (like `requireTenant` invoking `isDogfoodingEmail` and `isOperatorEmail`) can cause unnecessary overhead on almost every authenticated API call. Simple memoization patterns might fail test suites if they don't account for dynamic environment changes like `vi.stubEnv`.
**Action:** When caching parsed environment variables (such as Sets or Arrays) used in middleware or frequently called functions, memoize the parsed result but invalidate the cache using a raw string comparison with the current `process.env.VAR` (e.g., `process.env.VAR === cachedRawVar`). This avoids parsing overhead per request while preserving correct testability.
