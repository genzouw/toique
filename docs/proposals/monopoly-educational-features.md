# Toique: LINEで学ぶリアルビジネスシミュレーション（モノポリー/いただきストリート機能の統合案）

## 概要

ToiqueのLINEチャットボットUIとマルチテナント・フォームエンジンの基盤を活用し、「現実のマネーゲームを模倣し、経済やビジネスを学ぶ教材」としての機能をToiqueのオプションサービス、あるいは新機軸のテンプレートとして導入する提案です。

従来の「モノポリー」や「いただきストリート」といったボードゲームの要素（不動産売買、株取引、増資）をLINE上で展開しつつ、SaaSや現代の金融システムなど、現実のビジネス要素を組み込んだ「教育的シミュレーション機能」を提案します。

## ターゲット層

- **企業研修:** 新入社員や若手社員向けのビジネス・経済の基礎研修
- **教育機関:** 中高生〜大学生向けの金融リテラシー教育
- **一般ユーザー（B2C）:** 楽しく経済を学びたい個人向け

## 基本機能（従来のモノポリー/いただきストリートのLINE最適化）

1. **LINEフォームエンジンを用いた「ターン進行」**
   - Toiqueのフォームエンジンを拡張し、サイコロを振る、マスのアクション（物件購入、株取引）を選択する、といった一連の行動をLINEのトーク画面で完結させます。
   - `trigger`キーワードでゲーム開始、`Quick Reply`で「サイコロを振る」「株を買う」「物件を増資する」などの選択肢を提示します。

2. **管理画面（ダッシュボード）との連携**
   - プレイヤーはLINEで操作し、ゲーム全体のマップ状況、各プレイヤーの総資産、株価の変動などは、Toiqueの管理画面（React + Tailwind UI）を拡張した「ゲーム観戦ダッシュボード」でリアルタイムに確認できるようにします。これにより、研修などで全員で画面を見ながら進行することが可能です。
   - ゲームの進行状態はサーバーサイドで一元管理し、WebSocketを介してダッシュボードへリアルタイムに配信します。

3. **基本システムの再現**
   - **エリアと物件:** エリアごとに物件を独占することで収益率アップ。
   - **株取引と相乗り:** エリアの株を買い、自分や他人がそのエリアの物件に増資すると株価が上がり、配当やキャピタルゲインを得られる（いただきストリートのシステム）。

## 現代・現実のビジネスを模倣した新機能案（従来では考えられない機能）

ここからが、単なるゲームのコピーにとどまらない、Toiqueならではの「現実のマネーゲームを学習する教材」としての拡張案です。

### 1. サブスクリプション・SaaSモデルの導入（MRRの概念）

- **従来のゲーム:** 止まった時だけ一過性の「レンタル料」を払う。
- **新機能:** 物件（例：クラウドサーバーエリア、SaaSプロダクトエリア）を購入すると、他プレイヤーが止まらなくても、**毎ターン継続的に少額の収益（MRR: 月次経常収益）が入る**。
- **学習効果:** サブスクリプションビジネスの強さ、LTV（顧客生涯価値）、チャーンレート（解約率：イベントカードなどでランダムに顧客が離れる）の概念を学べる。

### 2. 人材採用と人的資本投資（給与と生産性）

- **従来のゲーム:** コマを進めるだけ。
- **新機能:** プレイヤーは「従業員（エンジニア、営業など）」を雇用できる。雇用すると毎ターン「給与（コスト）」が発生するが、特定のマスでの収益性が倍増したり、サイコロの目が有利になったりする。
- **学習効果:** 人件費はコストだけでなく「投資」であること、ROI（投資対効果）の計算を体感できる。

### 3. マクロ経済イベントのリアルタイム連動（API連携）

- **従来のゲーム:** チャンスカードなどのランダムイベント。
- **新機能:** Toiqueのバックエンドから現実の株価指数（日経平均など）や為替レート、あるいは金利APIを定期的にフェッチし、ゲーム内の「好景気/不景気」や「資金調達コスト（借入金利）」に連動させる。
- **学習効果:** 現実のニュースやマクロ経済が、個別のビジネス（ゲーム内の資産）にどう影響するかをリアルタイムに感じられる。

### 4. クラウドファンディングとM&A（プレイヤー間交渉）

- **従来のゲーム:** シンプルな物件交換や競売。
- **新機能 (M&A):** 他のプレイヤーの会社（物件群）を敵対的買収（TOB）、あるいはLINEのトーク上で交渉して友好的買収ができる。Toiqueの管理画面から契約（合意形成と自動的な資産・資金の移転処理）を行う。
- **新機能 (クラウドファンディング):** 大きな物件を買う際、LINEグループ上で他のプレイヤーにプレゼン（メッセージ送信）し、資金を集めて共同所有する。利益は出資比率で分配。
- **学習効果:** 資金調達の多様性、交渉術、企業価値評価（バリュエーション）を学べる。

### 5. 複式簿記と財務諸表の自動生成（教育的フィードバック）

- **従来のゲーム:** 「総資産」という単一の数字しか見えない。
- **新機能:** プレイヤーの全トランザクション（LINEでの選択結果）をToiqueのバックエンドで複式簿記として記録し、ターンごとに「B/S（貸借対照表）」「P/L（損益計算書）」「C/F（キャッシュフロー計算書）」を自動生成してLINEで通知（PDF化して送信なども可）。
- **学習効果:** 「なぜ黒字倒産するのか（キャッシュフローの重要性）」「資産と負債のバランス」といった財務の基礎を、自分のプレイ履歴から自然に読み解けるようになる。

## 実装ステップ（Phase案）

- **Phase 1: 基礎エンジン拡張**
  - Toiqueのフォームエンジンを拡張し、ゲーム全体の進行状態（プレイヤーの資金、所有物件、ターン数など）を管理する「ゲームステート管理機能」を導入する。
  - LINE経由でのサイコロと簡単な物件購入フローの実装。
  - 財務諸表の自動生成（Phase 3）を見据え、複式簿記に対応したトランザクションスキーマを当初から設計する。
- **Phase 2: 株取引・ダッシュボード開発**
  - 株の概念追加、価格変動ロジックの実装。
  - 管理画面に観戦用ダッシュボード（WebSocket等でのリアルタイム更新）を追加。
  - 株価変動アルゴリズムには、流動性制約やボラティリティ上限などのマーケット操作防止策を組み込む。
- **Phase 3: 現実拡張要素の導入**
  - B/S, P/Lの自動生成・通知機能。
  - サブスクモデル、人材雇用などの新ルール追加。

## 結論

Toiqueの強みである「LINEという身近なインターフェース」と「堅牢な管理画面」を組み合わせることで、従来のボードゲームよりも圧倒的にリアルで、教育効果の高いビジネスシミュレーションツールを提供できます。
